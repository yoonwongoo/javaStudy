여러 알고리즘을 하나의 추상적인 접근점(Interface)을 만들어 접근점에서 서로 교환 가능(Delegate)하도록 하는 패턴
사용자(Client)는 자신에게 맞는 전략(Strategy)을 선택하여 로직을 수행할 수 있게하는 방법입니다.
예로들면 학생은 공부라는 작업을 수행함에 있어 공부를 상황에 맞게 선택해서 공부를 위임할 수 있습니다.

장점은 학생은 얼마든지 기존의 코드를 변경하지않고 하고 싶은 공부를 더 추가하여서 할 수 있다,
ex)-> 현재 작성되어있는 코딩,영어공부를 제외하고 국어공부, 수학공부를 더 추가할 수 있다.
